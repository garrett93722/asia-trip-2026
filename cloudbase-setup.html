TENCENT CLOUDBASE

国内可访问的投票同步

V7已经写入环境ID asia-trip-2026-d7g1rngaqcd47c8fb，不再使用CloudBase。只需要开启匿名登录、创建一个集合、设置规则并部署网站。

最终只创建一个集合：tripVotes。方案票、景点票和留言都存在这个集合中。
1

开启匿名登录

  1. 进入当前CloudBase环境。
  2. 打开“身份认证 / 登录方式”。
  3. 找到“匿名登录”，点击启用。
2

创建数据库集合

  1. 进入“数据库 / 文档型数据库”。
  2. 点击“新建集合”。
  3. 集合名称必须填写 tripVotes，大小写完全一致。
  4. 创建后无需手工添加记录。
3

设置集合安全规则

打开 tripVotes 的权限或安全规则，切换到自定义安全规则，粘贴：

{
  "read": true,
  "create": "auth.uid != null",
  "update": "auth.uid != null",
  "delete": false
}

规则含义：所有人可以查看结果；完成匿名登录后可以新增或修改投票；网页端不能删除记录。

4

部署到CloudBase静态托管

  1. 进入“静态网站托管 / 网站部署”。
  2. 选择“本地项目部署”。
  3. 上传V7解压后的文件夹，或上传本压缩包。
  4. 确保部署根目录直接包含 index.html 和 vote.html。
  5. 等待状态显示成功，点击“访问应用”。

系统会分配一个 *.tcloudbaseapp.com 测试域名。四个人先直接使用这个地址即可。

5

两台设备测试

  1. 打开新域名下的 /vote.html。
  2. 顶部应显示“腾讯云已连接”。
  3. 电脑输入一个姓氏并投票。
  4. 手机打开同一地址，应看到票数变化。
  5. 再测试一条景点留言。
6

报错排查

  • 匿名登录失败：回身份认证确认匿名登录已启用。
  • DATABASE_NOT_FOUND：确认集合名是 tripVotes。
  • PERMISSION_DENIED：确认集合规则已经保存并生效。
  • 域名/CORS错误：在环境安全来源或Web安全域名中加入CloudBase分配的访问域名。
  • 仍显示旧页面：静态托管控制台上传后会自动刷新CDN；也可给地址追加 ?v=20260718-cb1。
