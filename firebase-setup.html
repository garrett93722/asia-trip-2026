FIREBASE SETUP

让四台手机看到同一份投票

GitHub Pages只能放网页文件，不能替你保存所有人的投票。这里用Firebase的匿名登录和Cloud Firestore保存结果。 整个过程通常只做一次，完成后大家继续使用同一个GitHub Pages地址。

开始前准备：用电脑登录Google账号，打开你的GitHub仓库 garrett93722/asia-trip-2026，并解压V6网站文件。
1

创建Firebase项目

  1. 打开 Firebase Console。
  2. 点击“创建项目”。
  3. 项目名填写 asia-trip-2026。
  4. Google Analytics可以关闭，然后完成创建。
2

注册Web应用并复制配置

  1. 进入项目首页，点击网页图标 </>。
  2. 应用昵称填写 trip-vote-web。
  3. 不要勾选Firebase Hosting，因为网页仍由GitHub Pages托管。
  4. 点击注册应用，页面会出现一段 firebaseConfig。
  5. 在电脑中打开V6的 firebase-config.js，把REPLACE_ME整段替换为控制台给你的值。
export const firebaseConfig = {
  apiKey: "控制台给你的值",
  authDomain: "控制台给你的值",
  projectId: "控制台给你的值",
  storageBucket: "控制台给你的值",
  messagingSenderId: "控制台给你的值",
  appId: "控制台给你的值"
};

这段网页配置不是管理员私钥，可以放在公开仓库；权限由后面的Firestore Rules控制。

3

开启匿名登录

  1. 左侧进入 Build → Authentication。
  2. 点击开始使用。
  3. 进入 Sign-in method / 登录方法。
  4. 选择 Anonymous / 匿名，启用并保存。

用户不需要注册账号。页面会在后台获得匿名身份，旅行成员仍以自己输入的姓氏显示。

4

创建Cloud Firestore

  1. 左侧进入 Build → Firestore Database。
  2. 点击创建数据库。
  3. 选择 Production mode / 生产模式。
  4. 区域选择一个离亚洲较近的可用区域，完成创建。
5

发布安全规则

  1. 在Firestore页面点击顶部 Rules / 规则。
  2. 打开V6文件夹中的 firestore.rules。
  3. 复制文件全部内容，覆盖控制台中的默认规则。
  4. 点击 Publish / 发布。

不要长期使用“任何人都能写入”的测试规则。本项目规则要求匿名登录，并限制身份长度、投票值和留言长度。

6

把V6上传到GitHub

进入仓库根目录，点击 Add file → Upload files。把V6解压后的全部内容上传并覆盖旧文件，尤其包括：

index.htmltrip-data.js vote.htmlvote-data.js vote.jsvote.css firebase-config.jsfirestore.rules firebase-setup.htmlassets/ 文件夹

滚动到页面底部，填写提交说明，例如 Update full trip and voting system，然后提交。

7

用两台设备测试

  1. 等待GitHub Pages重新部署。
  2. 打开 https://garrett93722.github.io/asia-trip-2026/vote.html。
  3. 页面顶部应显示“云端已连接”，而不是“本地预览”。
  4. 手机A输入一个姓并投票。
  5. 手机B或电脑无痕窗口打开同一地址，应能立即看到票数变化。
  6. 再测试一条景点留言。
完成后打开投票页
8

常见问题

  • 仍显示本地预览：检查 firebase-config.js 是否仍有REPLACE_ME，或浏览器控制台是否报错。
  • 提示权限不足：检查匿名登录是否启用，以及Firestore规则是否点击了发布。
  • 修改后页面没变化：等待1–3分钟，再强制刷新；也可在GitHub Actions查看部署状态。
  • 同姓覆盖：页面按身份文字覆盖同一人的选择。若有同姓，改成“杨1”“杨2”或昵称。
  • 隐私：不要在留言中填写证件号、订单号、电话或精确住址。
